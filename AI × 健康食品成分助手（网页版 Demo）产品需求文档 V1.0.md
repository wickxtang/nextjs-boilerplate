### 1. 产品定位与核心逻辑

**一句话介绍**：
这是一款帮助用户**看懂配料表背后风险**，记录每日饮食并建立**配料-食物双链图谱**的健康工具。

**底层逻辑整合**：
- **风险维度**：WHO/IARC 致癌物等级（1类/2A/2B/3类）标注。
- **记录维度**：日历化打卡，支持实时/补录每日摄入的零食。
- **统计维度**：周期性汇总摄入最高频的配料与食物，发现潜在隐患。
- **关系维度**：以 **“配料成分”为节点**，打通用户所有记录的零食，建立可视化关联图谱。

**核心用户场景**：
> 用户拍下零食包装 → Gemini AI 毫秒级解析配料表 → 风险提示与营养解读 → 存入食物库 → 每日点击打卡 → 自动生成周/月健康统计报告 → 查看配料图谱发现成分关联。

### 2. 功能模块详细设计

#### 模块一：AI 配料表智能解析（首页）

**2.1 输入交互**
- **双图模式**：左侧上传/拍摄食物外观图，右侧上传/拍摄配料表图。
- **AI 智能识别**：调用 **Google Gemini 1.5 Flash** 视觉模型，自动从图片中提取配料文字并转化为标准化列表。

**2.2 处理逻辑**
- **实体提取**：自动去除无关标点，提取核心配料名称。
- **风险分类与解读**：AI 根据预设的致癌物库自动分级（红/黄/蓝），并给出通俗易懂的健康建议。

#### 模块二：日历打卡与健康统计（分析页）

**2.3 饮食日历**
- **可视化打卡**：日历视图展示每日打卡状态，绿色圆形表示已打卡，橙色圆形表示当天。
- **补录功能**：支持选择食物库内的存量食物，回溯记录过往日期。

**2.4 健康汇总报告**
- **近一周/一月统计**：自动计算该周期内出现次数最多的 5 种零食及 5 种核心配料，通过背景色区分风险等级。

#### 模块三：个人双链零食库（图谱页）

**2.5 搜索与管理**
- **全局搜索**：支持在“我的食物库”和“所有食物库”中按名称或配料快速检索。
- **编辑/删除**：用户可随时修正 AI 识别错误的配料或更新食物图片。

**2.6 全域图谱视图**
- 使用 ECharts 渲染，配料与零食作为不同颜色的节点，通过连线展示从属关系，支持拖拽、缩放及点击跳转。

#### 模块四：健康知识库（百科页）

**2.7 权威分级检索**
- 详细说明 IARC 各类致癌物的定义与建议，支持关键词快速搜索特定成分。

### 4. 技术栈实现

| 角色 | 技术选型 | 核心原因 |
| :--- | :--- | :--- |
| **框架** | **Next.js 14** | 全栈开发，API Routes 处理服务端逻辑，App Router 路由管理。 |
| **AI 接口** | **Gemini 1.5 Flash** | 视觉识别（OCR）与语义分析二合一，中文支持出色且响应极快。 |
| **数据库** | **SQLite (LibSQL)** | 轻量级、无服务器化，适合个人 Demo 阶段的数据持久化。 |
| **动效** | **Framer Motion** | 提升页面切换、列表加载及搜索交互的灵动感。 |
| **图表** | **ECharts** | 强大的 Graph 关系图支持。 |
| **UI** | **React Calendar** | 快速实现打卡交互。 |
